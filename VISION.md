# SDP Vision

SDP — AI-managed platform for the full software development lifecycle.

From idea to deployed feature through structured AI phases, agents, and verifiable gates.

## Two Phases

**Discovery** — research before you build:
- Frame the problem (JTBD canvas)
- Formulate hypotheses (RAT ranking)
- Scan the landscape (7 source types)
- Validate with evidence (desk research)
- Verdict: GO / PIVOT / KILL

**Delivery** — build with enforcement:
- Phase FSM: Discover → Plan → Build → Review → Eval
- Gates at each transition (human + automated)
- Evidence from real tool outputs — not self-report
- Gate fail = stop, not bypass

## The Promise

You drop an idea in → Discovery agents research and shape it → you approve the scope → Delivery agents implement through structured phases → the feature ships with a verifiable evidence trail.

## What SDP Is Not

- Not just CI/CD (no agent phases, no gate enforcement)
- Not just governance (no implementation assistance)
- Not just an agent runner (no structured phases, no evidence)
- Not a replacement for your IDE or harness (integrates with Claude Code, Cursor, Codex)
