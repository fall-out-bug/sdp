# WS: Safe Defaults Configuration

**Feature**: F076 - Guided Onboarding Wizard
**ID**: 00-076-04
**Priority**: P0
**Depends on**: 00-076-01

## Goal
Define and implement safe defaults for each project type that can be used in auto mode and as fallbacks in interactive mode.

## Scope Files
- `sdp-plugin/internal/sdpinit/defaults.go` (NEW)
- `sdp-plugin/internal/sdpinit/defaults_test.go` (NEW)

## Acceptance Criteria
1. Safe defaults defined for go, node, python, mixed, unknown project types
2. Defaults include: skills, validators, evidence settings, documentation paths
3. GetDefaults(projectType) returns appropriate defaults
4. Defaults can be overridden via configuration
5. Tests for all project types (>= 80% coverage)

## Out of Scope
- Interactive prompts (00-076-02)
- Auto mode (00-076-03)

## Implementation Notes
```go
type ProjectDefaults struct {
    Skills          []string
    Validators      []string
    EvidenceEnabled bool
    DocPath         string
    TestCommand     string
    BuildCommand    string
}

func GetDefaults(projectType string) *ProjectDefaults {
    switch projectType {
    case "go":
        return &ProjectDefaults{
            Skills: []string{"feature", "idea", "design", "build", "review"},
            TestCommand: "go test ./...",
            BuildCommand: "go build ./...",
        }
    // ... other types
    }
}
```

## Test Strategy
- Unit tests for each project type defaults
- Verify defaults are valid and usable
- Test fallback for unknown types
