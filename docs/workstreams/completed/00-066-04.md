## CLI Dependencies

SDP follows the principle: **prompts > hooks > go module**

### Dependency Levels

| Level | Description | Install Required |
|-------|-------------|------------------|
| **NONE** | Pure prompts, no CLI needed | No |
| **OPTIONAL** | CLI enhances functionality | No (but recommended) |
| **REQUIRED** | CLI is mandatory | Yes |

### Quick Reference

| Feature | CLI Level | Fallback |
|---------|-----------|----------|
| @build | OPTIONAL | Shell scripts |
| @review | OPTIONAL | Reduced validation |
| @oneshot | OPTIONAL | Shell scripts |
| Contract validation | REQUIRED | None |
| Memory search | REQUIRED | None |
| Evidence chain | REQUIRED | None |

# Just copy prompts (no Go required)

# Getting Started: Prompts Only

Want to try SDP without installing Go? Here's how.

## What Works Without CLI

✅ Core skills: @build, @review, @oneshot, @feature
✅ Git safety checks (basic)
✅ Workstream management
✅ Quality gates (via shell)

## What Requires CLI

❌ Contract validation
❌ Memory/search system
❌ Evidence chain
❌ Advanced drift detection

## Installation

1. Clone SDP repo
2. Copy prompts to your project:
   ```bash
   cp -r sdp/.claude/ your-project/.claude/
   cp -r sdp/prompts/ your-project/prompts/
   cp -r sdp/scripts/ your-project/scripts/
   ```
3. Done! Start using skills.

## Limitations

Without CLI, you get:
- Basic git safety (branch checks)
- Simple drift detection (file existence)
- No persistent memory
- No contract validation

For full functionality, install the CLI:
```bash
go install ./sdp-plugin
```
```
