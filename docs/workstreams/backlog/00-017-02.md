# 00-017-02: working-tree context packet and test evidence

## Goal

Build the deterministic context packet for `sdp pi-review` scopes.

## Acceptance Criteria

- [ ] AC1: `--scope working-tree` includes staged, unstaged, and untracked reviewable files.
- [ ] AC2: `--scope branch --base <ref>` uses `git diff <base>...HEAD` and includes touched file content.
- [ ] AC3: `--scope auto` uses PR metadata when available, otherwise falls back to working-tree review.
- [ ] AC4: The packet records exact reviewed files plus hashes for diff, rules, touched files, and test evidence.
- [ ] AC5: In delivery mode, missing or failing deterministic test evidence prevents `APPROVED`.

## Context

Models should not run tests independently in MVP. The CLI owns one deterministic test command and gives the output to model reviewers as evidence.

## Dependencies

- 00-017-01: schema and spec names must be stable before implementation.

## Steps

1. Add the `sdp pi-review` command skeleton with `--scope`, `--base`, `--feature`, `--create-beads`, and `--write-verdict` flags.
2. Implement reviewed-file discovery for working-tree and branch scopes.
3. Implement context packet serialization under `.sdp/runs/pi-review/<run_id>/packet.json`.
4. Resolve review command from `.sdp/review.yaml`, then `.sdp/config.yml`, then conservative local fallback.
5. Add tests for dirty working tree, untracked file inclusion, branch diff, and missing test evidence.

## Expected Result

`sdp pi-review` can produce a complete packet and test-evidence artifact without invoking `pi` yet.

## Constraints

- Do not silently ignore dirty files.
- Do not fail on absent PR metadata when working-tree scope is available.
- Do not send files outside the reviewed file list to model reviewers.
