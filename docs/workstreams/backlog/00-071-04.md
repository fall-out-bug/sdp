## Goal

Expose team-relevant state changes through dashboard summaries and lightweight notifications.

## Context

Teams need shared visibility without requiring everyone to inspect raw logs continuously.

## Acceptance Criteria

- [ ] AC1: Define minimum dashboard fields (active WS, blocked WS, pending approvals)
- [ ] AC2: Add notification hooks for critical workflow events
- [ ] AC3: Ensure notifications include direct remediation links/commands
- [ ] AC4: Add configuration docs for notification channels
- [ ] AC5: Validate no leakage of sensitive content in notifications
