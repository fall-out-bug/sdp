## Goal

Establish a deterministic baseline of repository health for agents, CI, release, and repository hygiene to anchor all F067 improvements.

## Context

F067 is a hardening feature. Without a frozen baseline, follow-up workstreams risk optimizing the wrong bottlenecks or moving targets.

This workstream creates the authoritative "before" snapshot used by subsequent WS validation. Data comes from two independent audits (PR #32 analysis + cloud agent analysis).

## Implementation Notes

### Baseline Inputs

Collect from:
- `go.mod` and `sdp-plugin/go.mod`
- `.github/workflows/*.yml`
- `sdp-plugin/Dockerfile`
- `prompts/*`, `.cursor/*`, `.opencode/*`
- `.golangci.yml` exclusion rules
- All non-test `.go` files `wc -l`
- git-tracked large/generated artifacts

5. Distribution contract consistency
6. Repository hygiene
