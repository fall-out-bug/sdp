# 00-017-04: verdict, telemetry, beads, and delivery-loop integration

## Goal

Persist pi-review outputs and make the delivery loop consume them as the review gate.

## Acceptance Criteria

- [ ] AC1: `--write-verdict` writes compact `.sdp/review_verdict.json` conforming to `schema/review-verdict.schema.json`.
- [ ] AC2: Raw model outputs and run telemetry are written under `.sdp/runs/pi-review/<run_id>/` and are gitignored by default.
- [ ] AC3: `--create-beads` creates or updates deduped findings with source, feature, workstream, run id, severity, blocking flag, evidence, recommendation, and source model ids.
- [ ] AC4: Delivery-loop guidance uses `build/fix -> sdp pi-review -> beads findings -> fix -> repeat until clean`.
- [ ] AC5: The loop has runaway escalation for wall-clock budget, repeated same finding, provider failure budget, and repeated non-convergence.

## Context

This is the integration slice. Review remains read-only; fix subagents own code changes.

## Dependencies

- 00-017-01: contract.
- 00-017-02: context packet.
- 00-017-03: model synthesis.

## Steps

1. Implement verdict writing and schema validation.
2. Implement raw artifact and run manifest writing.
3. Add `.gitignore` rules for raw pi-review artifacts while keeping compact verdict behavior unchanged.
4. Implement beads create/update with dedupe hash.
5. Update review/delivery-loop skill guidance to call `sdp pi-review`.
6. Add integration tests for `--create-beads`, `--write-verdict`, clean review, blocking review, and escalation.

## Expected Result

An autonomous delivery run can repeatedly call `sdp pi-review`, receive beads findings, fix them, and converge without a separate Codex-only review phase.

## Constraints

- Do not auto-fix findings in the review command.
- Do not silently skip pi-review in delivery mode.
- Do not commit raw model transcripts by default.
