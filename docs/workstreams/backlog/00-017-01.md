# 00-017-01: pi-review contracts and schemas

## Goal

Lock the public contract for the working-tree-first `sdp pi-review` gate.

## Acceptance Criteria

- [ ] AC1: `docs/reference/pi-review-spec.md` defines command surface, scope model, context packet, test evidence, model policy, synthesis, artifacts, beads integration, and delivery-loop behavior.
- [ ] AC2: `schema/pi-review-run.schema.json` validates the run telemetry manifest without storing raw model text.
- [ ] AC3: `schema/review-verdict.schema.json` accepts external review runtime metadata while remaining compatible with existing `@review` verdicts.
- [ ] AC4: `schema/index.json` and reference indexes link the new contract.

## Context

This workstream is the protocol slice. It must not implement the CLI runner. The output is a stable contract that later implementation work can test against.

## Dependencies

None.

## Steps

1. Review `docs/reference/pi-review-spec.md` against the F017 decision: working tree is canonical, PR metadata is optional, and `sdp pi-review` is review-only.
2. Validate the JSON schemas listed in `verification_commands`.
3. Confirm the `review-verdict` additions are optional fields, not new required fields.
4. Update wording if any field name disagrees with implementation workstreams before coding starts.

## Expected Result

The repository has a review-gate contract clear enough for an implementation agent to build without re-litigating scope.

## Constraints

- Do not hard-code GLM/Kimi model ids in schemas or docs.
- Do not put raw transcripts into `.sdp/review_verdict.json`.
- Do not make `pi-review` responsible for code fixes.
